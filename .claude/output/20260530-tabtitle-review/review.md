# Skeptical review — tab-title-badge.md (IPC "push rung" via Ghostty tab badge)

Target: `/Users/alcatraz627/Code/Claude/claude-ipc/docs/notes/tab-title-badge.md` (UNBUILT proposal).
Stance: adversarial. Every row cites a file:line I opened. Bias: surface anything ≥40%.

**Headline:** the proposal is self-defeating for its stated purpose. The badge only
updates on the session's OWN Stop hook, but the target use case ("ask an idle
session") is BY DEFINITION a session that is not taking turns — so its Stop hook
never fires and its badge never updates while idle. The project's own architecture
docs already say this case needs `--channels` (`02-behavior.md:105-110`,
`03-architecture.md:82-85`). The note even half-admits it in its own open question
(`tab-title-badge.md:53-54`) but proceeds anyway. This is the same gap re-skinned,
not a fix for it.

## Bottom line for the human

1. **Kill or pivot.** As written (badge on the session's own Stop hook), the
   mechanism cannot signal an idle tab — the only case it exists for. It is the
   turn-boundary rung wearing a costume. Cited: `02-behavior.md:105-110`,
   `03-architecture.md:79-85`, `Stop.tasks:5`.
2. **The only non-dead variant** is broker-writes-to-peer-TTY (70% row): the
   broker holds each alias's cached `TTY_PATH` and emits the OSC escape to *that*
   pty out-of-band. That is unproven (can a daemon write another session's
   `/dev/ttysNNN` and have Ghostty honor it?) and is functionally a hand-rolled
   `--channels`. Prototype that one cross-tab write before designing anything else.
3. **Two stated primitives don't exist yet:** the "cheap count op" (`router.ts`
   has only full-payload `check`) and any alias↔session↔TTY join. Both are
   prerequisite, neither is in the note's plan.
4. **Slot collision is real:** the composer re-emits the whole title every turn
   (`update-tab-title.sh:113`); any independent IPC writer is clobbered. Needs a
   first-class slot or a decorator — but a decorator doing a socket call breaks
   the documented "no network" decorator contract (`features/tab-title.md:167`).
